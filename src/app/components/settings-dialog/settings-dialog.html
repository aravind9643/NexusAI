🦙 Ollama (Local AI)

To use Ollama from this web interface, you must enable CORS permissions on your machine:

Windows (PowerShell): 
[Environment]::SetEnvironmentVariable("OLLAMA_ORIGINS", "*", "User")
Restart Ollama from the system tray after running this.
Mac / Linux: 
launchctl setenv OLLAMA_ORIGINS "*"
Or run: OLLAMA_ORIGINS="*" ollama serve
Mixed Content Note: If the app is on HTTPS (like Vercel) and Ollama is on HTTP (localhost), your browser will block the request. You must click the Site Settings (lock icon) in the URL bar and set Insecure Content to Allow.

💻 LM Studio

Ensure the Local Server is running and CORS is enabled:

  1. Go to the Local Server tab in LM Studio.
  2. Check the CORS setting in the right-hand sidebar (set to "ON" or "*").
  3. Restart the server and click Test in NexusAI.
Don't forget to load a model in LM Studio before starting the server!

🌐 Cloud Providers

OpenRouter: Best for accessing Claude, GPT-4, and Gemini with one key. Base URL: https://openrouter.ai/api/v1
Groq: Ultra-fast inference. Base URL: https://api.groq.com/openai/v1
OpenAI: Direct GPT access. Base URL: https://api.openai.com/v1

⚡ Quick Troubleshooting

  • Models not showing? Click the refresh icon in the model selector.
  • Connection Error? Check if your API key is correct or if Ollama is running.
  • Slow responses? Local speed depends on your GPU/CPU power.
